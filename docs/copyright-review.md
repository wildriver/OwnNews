# OwnNews 著作権・権利関係の整理（2026-07 調査）

## 前提

- ニュースソースは CEEK.JP NEWS（RSS）。**CEEKからは利用許可を取得済み**。
- ただし、CEEKの許可がカバーするのは「CEEKのサービスの利用」であり、
  記事の見出し・概要・写真の著作権は**元の報道各社に帰属**する。
  CEEKは各社の再許諾権を持たないため、各社との関係は権利制限規定で適法である必要がある。
- 戦略: **CEEKと同等以下の態様（見出し＋短い抜粋＋原典誘導）に揃える**ことで、
  ニュース検索サービスとして確立した適法性のラインに立つ。

## 法的な足場

1. **著作権法47条の5（所在検索・情報解析サービスの軽微利用）** — 中核的根拠。
   「媒体名・日付・見出し・ごく短いスニペット・URL・サムネイル」の表示を許諾なしで認める。
   - 日本新聞協会の解釈（2021年12月 Q&A）: スニペットは本文の**5〜10%程度**、
     リード文全部はNG、サムネイルは「それと確認できる程度」、原典リンク誘導が必須、
     サービス内で完結して原典の代替になるものはNG。
   - 施行令・施行規則の要件: robots.txt等の収集拒否の尊重、ID/パスワード管理記事の不使用、
     **問い合わせ窓口の明示**、要件解釈の確認等の取組。
2. **30条の4（情報解析・非享受利用）** — 埋め込み生成・栄養素スコア・分類・推薦計算は明確に適法。
3. **判例**
   - YOL事件（知財高裁2005）: 見出し自体は非著作物だが、見出しのみの大量・反復・営利配信は不法行為になり得る。
     → 原典誘導型の本サービスはYOL型と異なる。
   - リツイート事件（最高裁2020）: 画像の自動トリミング表示が氏名表示権侵害。
     → 画像はクロップ表示を避け、出典を明記する。

## 機能別評価と実装対応（2026-07-10 実施）

| 機能 | 評価 | 対応 |
|---|---|---|
| 見出し・媒体名・日付・原典リンク | 低リスク | 47条の5の想定要素。変更なし |
| 埋め込み・栄養素・分類・推薦 | 低リスク | 30条の4。変更なし |
| 概要（スニペット） | 中 → 対応済み | 公開パック・詳細ページとも**120字**に短縮＋「続きは配信元で」誘導を明記（解析用の内部利用は300字のまま=30条の4） |
| 記事詳細の画像 | 高 → 対応済み | 大型表示を廃止し**小さめサムネイル（h-40・クロップなしのobject-contain）＋「画像: 配信元（媒体名）」表記**に。カードの小サムネイルは「確認できる程度」として維持 |
| AI深掘り解説 | 中 → 対応済み | システムプロンプトで**記事の要約・引用・再現を禁止**し「記事に書かれていない背景・文脈」に限定。UIに「記事の要約・転載ではない」注記を表示 |
| 問い合わせ窓口（省令要件） | 必須 → 対応済み | `/about` に運営者情報・連絡先・**報道機関向けの掲載停止依頼窓口**を明示。LPフッター・サイドバーから導線 |
| 研究目的の明示 | 推奨 → 対応済み | LP・/about に「大学研究室による研究プロジェクト・非営利」を明記（ただし書き判断・不法行為判断で有利） |
| はてなブックマークコメント | 低リスク | 公式提供の無料API・出典リンク表示済み |
| X連携 | 低リスク | リンク/インテントのみで記事本体の利用なし |

## 残タスク・運用上の注意

- [ ] CEEKの許可範囲（RSSの機械的利用か、summary・画像URLの再表示まで含むか）を書面等で明確化
- [ ] 報道機関から掲載停止依頼が来た場合の手順: 該当媒体をcollectorの除外リストに追加 → パック再生成
- [ ] 新機能で記事本文の表示量を増やす変更をする際は、この文書の「5〜10%」基準に立ち返る
- 収集はCEEK RSS経由のみ（各社サイトへの直接クロールなし）。robots.txt問題は生じない構成を維持する
